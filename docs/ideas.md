# Questflow — must-have идеи

Краткий бэклог продуктовых фич, которые стоит довести до «обязательного» уровня. Учитывает текущий код: воркспейсы и доски, геймификацию на карточках, профиль/настройки, социалку и безопасность.

**Связанные планы:** [gamification-roadmap.md](gamification-roadmap.md), [profile-settings-roadmap.md](profile-settings-roadmap.md), [habitica-mechanics-reference.md](habitica-mechanics-reference.md).

---

## Воркспейсы и совместная работа

1. **Поиск по воркспейсу** — карточки, комментарии, исполнители; один инпут с фильтрами по доске/списку.
2. **Метки и фильтры на доске** — цветные labels, быстрый фильтр «мои / просроченные / без исполнителя».
3. **Напоминания по дедлайну** — в разделе уведомлений `UserSettings.site.notifications`.
4. **Шаблоны досок** — Kanban «разработка», «бэклог», пустая; создание доски из шаблона в один клик.
5. **Архив досок и списков** — скрыть без удаления данных; восстановление для OWNER/ADMIN.
6. **@упоминания в комментариях** — уведомление адресату; связь с in-app центром уведомлений.
7. **Журнал активности v2** — фильтры по типу события, без экспорта CSV.
8. **Роли и делегирование** — явные права: кто может приглашать, удалять доску, менять billing-поля (если появятся).
9. **Онбординг пустого воркспейса** — подсказка «создай доску → список → карточку» для новых команд.

**Статус:** v1 реализован (2026-06-04): backend + UI; миграция `20260604120000_workspace_features`.

## Геймификация (прогрессионный слой)

13. **In-app центр уведомлений** — XP, квест завершён, сундук, ачивка, приглашение в рейд; читабельная лента + «прочитано».
14. **Связка email ↔ in-app** — те же тогглы, что в `/settings/notifications`; дубли только где уместно (security — email, квесты — in-app).
15. **Недельный чекин или weekly-квест-бандл** — +200 XP раз в ISO-неделю; не дублировать с дневным чекином.
16. **UI статуса HP = 0** — косметика «истощён» без блокировки досок; баннер «вчера не было активности — завтра −5».
17. **Лидерборд воркспейса (opt-in)** — топ XP за 7 дней среди участников WS; выключен по умолчанию, privacy в настройках WS.
18. **Battle pass lite (сезон 4 недели)** — трек наград (косметика/пыль), прогресс от квестов и активности, без pay-to-win.
19. **Расширение квестов** — шаблоны под комментарии, streak, party-рейд; прогресс в той же транзакции, что и событие.
20. **Party-рейды v1.1** — авто-kick AFK 2 дня, второй параллельный рейд (лимит), уникальный loot table на босса.
21. **E2E-регрессия геймификации** — сброс лимита 5 XP, идемпотентность карточки, cron HP, квест → сундук (см. Phase 4 roadmap).
22. **Единый `GAME_DAY_TZ` в UI** — подпись «игровые сутки до …» на профиле персонажа и в гайде.
23. **Интеграция привычек → XP** — после блока «Привычки»: закрытие Daily/To-Do/Habit+ даёт XP с отдельными лимитами и типами `XpEvent` (не ломая лимит карточек).

## Профиль, приватность, настройки

24. **Публичный @username или slug** — стабильная ссылка `/profile/@nick` вместо только numeric id (редирект со старого id).
25. **Bio на `/profile/me`** — 1–2 строки, видимость: все / друзья+коллеги / только я.
26. **Синхронизация темы в аккаунте** — light/dark в `UserSettings`, не только localStorage.
27. **2FA TOTP** — включение в security, backup codes, обязательный re-login при смене пароля с 2FA.
28. **Google: привязка / отвязка** — UI в security без потери единственного способа входа.
29. **Блокировка пользователя** — не видеть профиль, не писать в DM, не приглашать в пати.
30. **Push-уведомления (Web Push)** — отдельные тогглы от email; минимум: assign, mention, friend request.
31. **Timezone в профиле** — отображение дедлайнов и «игрового дня» по выбору пользователя (серверный TZ остаётся для cron).
32. **Canonical redirect legacy URL** — `/profile/user/:id` → `/profile/:id` через `replaceState` (роутер уже dual-match).

## Безопасность и доверие

33. **Аудит чувствительных действий** — смена email, пароль, 2FA, удаление аккаунта, revoke all sessions — в security log с IP/UA.
34. **Rate limits по сценариям** — invite spam, reset password, friend code brute; понятные коды API.
35. **Session anomaly (опционально)** — email security при входе с нового устройства/страны (уже есть тоггл — расширить эвристику).
36. **Workspace: запрет приглашений вне домена** — allowlist email domain для корпоративных WS.
37. **Ротация refresh и отзыв** — уже есть; must-have: UI «это устройство» на всех клиентах + предупреждение при revoke текущей.

## Привычки и solo-режим

Отдельный продуктовый слой для пользователей **без команды**: личные привычки, ежедневки и todo, с тем же персонажем и экономикой XP/HP. Не дублирует воркспейс — дополняет его.

1. **Личный хаб `/me` или `/habits`** — вход в solo-режим с дашбордом «сегодня»: Dailies, Habits, To-Do; CTA «создать воркспейс» для тех, кто созрел для команды.
2. **Три типа задач (как в Habitica-lite)** — **Habit** (+/−), **Daily** (расписание по дням недели), **To-Do** (разовые, без урона за просрочку).
3. **Игровой день и cron** — в полночь `GAME_DAY_TZ`: невыполненные Dailies → −HP (мягко, как на карточках); опционально «догнать вчера» (RYA) один раз.
4. **Награды за выполнение** — XP с лимитами по типу (например 5 habit+/daily/todo в сутки в сумме или раздельные квоты); +HP по тем же правилам, что `addExperience`.
5. **Сложность задачи** — trivial / easy / medium / hard множит XP; без pay-to-win статов.
6. **Streak на Daily** — серия дней, бонус XP на 7/14/30 (переиспользовать логику `CHECKIN_STREAK` или отдельный тип события).
7. **Личные Rewards** — трата «золота» или пыли на кастомные награды («час игры», «кофе»); gold как валюта или только dust/косметика в v1.
8. **Приватность по умолчанию** — привычки видны только владельцу; опционально «показать streak друзьям».
9. **Квесты solo** — дневные/недельные шаблоны «3 daily», «5 habit+», «закрыть 2 todo»; сундуки как на командных квестах.
10. **Интеграция с персонажем** — один `Character`; вкладка на `/profile/character`: «Сегодня в привычках» + ссылка в хаб; party-рейд по-прежнему от друзей, не от WS.
11. **Мобильный quick-complete** — свайп/тап +/- на habit и чекбокс daily без открытия полной формы.
12. **Импорт / шаблоны** — стартовый набор («Утро», «Спорт», «Вода») при первом входе в habits.
13. **Не смешивать с карточками WS** — отдельные таблицы `Habit`, `Daily`, `PersonalTodo`; общий только слой наград в `character.service`.
14. **Онбординг solo** — регистрация → создать персонажа → 3 задачи habits → первый XP; воркспейс не обязателен.
